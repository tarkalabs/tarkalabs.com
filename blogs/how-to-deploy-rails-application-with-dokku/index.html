Deploying your Rails application using dokku

There are several ways to deploy a Rails application. Couple of most commonly used ways are to the Heroku or your own VPS. The former promises to save time over money and latter the reverse.

As you might know, Heroku, a PaaS, is known for making deployment as seamless as possible. Git push into your Heroku app and you’re done. Whereas deploying to your own VPS is cheap and offers you customization, but involves a complicated process to get it up and running.

What if you could get the simplicity of the deployment that Heroku offers and the freedom from your own servers? dokku aims to be the answer.

In this post, we’ll see how to deploy an example Rails application to dokku.

What is dokku?

dokku is a PaaS implementation. Or much rather, an open source Heroku. So it’ll be like any other software that you install on your server to immediately grant it capabilities of a Platform. Since dokku is also a small implementation, you wouldn’t have much trouble using it in small servers.

Apart from its small implementation, dokku is also easy to extend and has support for many platforms for easy integrations. Stuff like adding SSL, notifications, auto-deployments are all easy to achieve in dokku. We’ll cover each of these individually in upcoming posts.

Initializing the app

Let’s now setup an example Rails app for us to deploy.

If you have an existing Rails app that you would like to follow this tutorial with, you can safely ignore this section.

We’re using the following versions for our sample app.

Rails: 6.0.32 Ruby: 2.7.1

Let’s initialize our Rails application with the following commands.

rails new dokku-rails --git --database=postgresql cd dokku-rails rake db:create && rake db:migrate

And let’s add some content to our landing page.

<h1>hello world</h1>
<p>Happy to say the mission is success.</p>

and, we’ll make it the default page as well. In config/routes.rb

Rails.application.routes.draw do root 'home#index' end

That’s it. Quickly verify everything’s right by starting your rails server and visiting [http://localhost:3000](http://localhost:3000/)

rails server

Let’s wrap things up by adding a git commit.

git add .
git commit -m "Initial commit"

Installing dokku on your server

Before we integrate dokku into our application, we’ll setup dokku on our server first. I’m using Hetzner for this exercise, but you can use a VPS provider of your choice. I’m provisioning a Ubuntu 20.0 server, but the instructions should be the same for most latest versions of Linux OS.

Before we start setting up, make sure to setup a deploy user to avoid using root user for deployments.

After you SSH into your server, let’s start by installing dokku.

wget https://raw.githubusercontent.com/dokku/dokku/v0.21.3/bootstrap.sh;
sudo DOKKU_TAG=v0.21.3 bash bootstrap.sh

This will install the 0.21.3 version of dokku and its dependencies on your server.

Once the installation is completed, head over to the dokku web admin by visiting http://<your-server-ip> to do a one-time configuration for your app. This is an important step to setup VHOST configurations and only after you set it up, will the web admin be stopped. Not doing it will cause the web-admin to continue existing and vulnerable.

The instructions are pretty straightforward here. What you have to do is, upload your local ssh public key which you can fetch using cat ~/.ssh/id_rsa.pub and set a host name. As you can see, I have selected the virtualhost and this is mostly preferential. If turned on, dokku will expose the apps via different port, else it'll be a subdomain based host.

All of these settings are changeable later, so don't worry if you made a mistake.

Now, dokku’s setup. Let’s get started with integrating into our application.

Integration with the app

Let’s create our dokku app on the server. SSH into your server, and run the following commands.

dokku apps:create dorails
sudo dokku plugin:install https://github.com/dokku/dokku-postgres.git
dokku postgres:create dorails_db
dokku postgres:link dorails_db dorails

We first start by creating the application in dokku. This will internally create appropriate Docker container and provision volumes and more for our application.

In the second step, we’re adding “backing services” for our application. For our sample app Postgres database is the only backing service we use. The dokku postgres plugin will take care of installing postgres and setting up the db users for us to use later.

The third step is where we’re creating the database to use in our application. You can specify any name you like, just make sure the database does not exist.

And in the final step, we link the created database with our application. This will essentially set a DATABASE_URL environment variable which will be picked up by Rails to connect to the database.

At the end of the last command, you’ll see a warning.

! App dorails has not been deployed

Let’s fix that.

First, switch to your local shell and cd into your project directory.

We’ll first have to create a Procfile in our application. Procfile is the manifest for our application that instructs dokku on the commands it has to execute to start our application and other services. Go to your project directory and create the Procfile at the root.

touch Procfile

and add the below content to the file.

web: bundle exec puma -C config/puma.rb

Here, we have declared a “web” service that will be started by running the declared command. In future posts, we’ll look into integrating other services. For now, we’ll stick with the web service. We’ll do a git commit now.

git add . git commit -m "Add Procfile"

Let’s now add our dokku server as a git remote so that we can push the app and do our first deployment!

git remote add dokku dokku@<server-ip>:dorails

Replace <server-ip> with your public IP of the server and the application name with the dokku app name you created in the previous step. Once added, kickstart your first deployment.

git push dokku master

Once the installation is completed, head over to your server’s IP address and voila.

Conclusion

That brings us to the conclusion of this post. We have now successfully deployed our Rails application to the internet using dokku. Anytime you make changes to your application, deployment is as easy as git add & git push to your dokku.

In future posts, we’ll look into adding custom domains for our dokku app, provisioning a SSL certificate and also explore some of dokku’s plugins.
