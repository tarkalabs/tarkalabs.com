Launching a screen within an app using custom Siri voice commands

Naren Krishna

iOS Developer

Navigation to the Specific Screen:

As an iOS developer, I attempted to integrate SiriKit to allow users to navigate to a specific screen inside an app using voice commands. I first considered using Intent and App extensions, which would require more effort to develop. Subsequently, I discovered that consolidating with NSUserActivity and IntentsUI is a piece of cake to open a specific app screen. Intent can increase interaction between Siri and your app, which is unnecessary in our case.

Let’s explore how easy it is to launch a specific screen using the Siri voice command.

  • Setting the voice phrase in NSUserActivity. (NSUserActivity is the object used to coordinate interactions between your app and Siri)
  • Connecting that to INUIAddVoiceShortcutViewController (a class file that helps to add a shortcut to Siri) in the screen which you wish to open
  • Navigation action

Add Shortcut ViewController using the IntentsUI framework:

We’ll provide the default phrase “Show nearby assets” in this example to tell SIRI to launch the screen that shows the nearby Assets. To achieve this, we’ll have to use the delegate methods provided by INUIAddVoiceShortcutViewControllerDelegate. Please refer screenshot below. However, in iOS 15, there is an option to remove the shortcut and change the phrase directly from the view if desired.

Configuring error messages

A method addVoiceShortcutViewController(_:didFinishWith:error:)will be called once the Done button is clicked. This is used to dismiss the AddVoiceShortcutViewController and display any error that has occurred. So, once you’ve added the shortcut, you can dismiss the view by hitting the Done button.

class SecondViewController: INUIAddVoiceShortcutViewControllerDelegate {

  override func viewDidLoad() {
    super.viewDidLoad()
  }

 func addVoiceShortcutViewControllerDidCancel(_ controller: INUIAddVoiceShortcutViewController) {
    dismiss(animated: true, completion: nil)
  }

  func addVoiceShortcutViewController(_ controller: INUIAddVoiceShortcutViewController, didFinishWith voiceShortcut: INVoiceShortcut?, error: Error?) {
    dismiss(animated: true, completion: nil)
  }
}

A set of delegate methods that will be called when the done and cancel buttons are pressed.

Set the suggestion phrase and title in NSUserActivity:

We have to provide the suggested shortcut phrase by using suggestedInvocationPhrase in NSUserActivity. So Let’s configure NSUserActivity to set a suggestion phrase, and we can also set the title of the Shortcut action. It will appear in the Shortcuts menu, as shown in the screenshot above.

extension NSUserActivity {

  public static let nearbyAssets = "com.naren.sirikit.showNearbyAssets"

  public static var showNearbyAssetsActivity: NSUserActivity {
    let userActivity = NSUserActivity(activityType: NSUserActivity.nearbyAssets)
    userActivity.title = "Displays all nearby assets in order to retrieve open work orders"
    userActivity.persistentIdentifier = NSUserActivityPersistentIdentifier(NSUserActivity.nearbyAssets)
    userActivity.suggestedInvocationPhrase = "Show nearby assets"
    return userActivity
  }
}

Let’s take a look what we have set here,

  • We have created an NSUserActivity instance by passing an Activity type value. You can set it based on your needs.
  • Title and Suggested Phrase values to show it in the AddVoiceShortcutViewController.
  • persistentIdentifier is used to clear the activity in the end.

We are almost done, So now we can connect the NSUserActivity setup to the INUIAddVoiceShortcutViewController class like the below code. This function should be called in the ViewController, which will be launched when the voice command is delivered in SIRI.

func addSiriShortcut() {
  let activity = NSUserActivity.nearbyAssets
  let shortcut = INShortcut(userActivity: activity)
  let addShortcutVC = INUIAddVoiceShortcutViewController(shortcut: shortcut)
  viewController.delegate = self
  present(addShortcutVC, animated: true, completion: nil)
}

Navigation to the Specific Screen:

We have to do a Final setup for Navigation to that Specific View Controller. Let’s use the AppDelegate method application(_:continue:restorationHandler:), which is used to handle activities from Siri based on the userActivity.

func application(_ application: UIApplication, continue userActivity: NSUserActivity, restorationHandler: @escaping ([UIUserActivityRestoring]?) -> Void) -> Bool {
  guard userActivity.activityType == NSUserActivity.nearbyAssets else {
    return false
  }

  guard let window = window, let rootvc = window.rootViewController as? UINavigationController else {
    return false
  }

  restorationHandler(rootvc.viewControllers)
  return true
}

We have included the Conditions and Navigation actions in this Method. This will open root VC, From that, we need to push the Specific View Controller. Let’s conclude this with our last function,

override func restoreUserActivityState(_ activity: NSUserActivity) {
  // Navigation action can be performed here based on the activity type
  if activity.activityType == NSUserActivity.nearbyAssets {
    let vc = ViewController()
    self.navigationController?.pushViewController(vc, animated: true)
  }
}

You can use this override method in the ViewController from where you need to navigate to the Specific screen. Based on the activityType you can perform the Navigation for other controllers also.

And yeah, you can start commanding SIRI to launch your App screens.
