The Code

You can find the source code I am building in this series over at github. Here’s the config module :

use anyhow::{Context, Result};
use config::{Config as RConfig, Environment, File};
use lazy_static::lazy_static;
use serde::{Deserialize, Serialize};
use std::env;

#[derive(Debug, Serialize, Deserialize)]
pub struct Database {
    pub url: String,
    pub max_connections: u32,
}

#[derive(Debug, Serialize, Deserialize)]
pub struct Config {
    pub host: String,
    pub port: i32,
    pub auth_token: String,
    pub database: Database,
}

impl Config {
    pub fn new() -> Result<Self> {
        let mut s = RConfig::default();

        s.merge(File::with_name("config/default"))
            .context("Unable to load the default config")?;

        let env = env::var("ENV").unwrap_or("development".into());
        s.merge(File::with_name(&format!("config/{}", env)).required(false))
            .context(format!("Unable to load config/{}.json", env))?;

        s.merge(Environment::new().separator("_".into()))?;

        s.try_into().context("Unable to instantiate Config struct")
    }
}

lazy_static! {
    pub static ref CONFIG: Config = Config::new().unwrap();
}

The Gist

The great thing about the library config is that it allows us to manage our configurations in layers allowing us to load configurations from multiple sources instead of just one. This gives us the advantage of logically dividing our configurations, allowing for easier management. You may have separate common configurations that apply to all environments and you may have separate environment specific configurations that are unique to each environment. You can also additionally override any configurations via the environment, which is useful when leveraging containers. This also helps when generating configurations on the fly.

Another aspect of configuration management is that you may want to build your configuration in a heirarchical manner instead of just a flat list of values. This also allows for easy overrides for sub-sections of the configuration.

Conclusion

Configuration management done right can really help you when you want to scale up your application and also when you need to setup multiple environments for your team. Automation is the key to keep things consistent and avoid issues and the above showcases how you can achieve that when managing configurations for an application built using Rust.
