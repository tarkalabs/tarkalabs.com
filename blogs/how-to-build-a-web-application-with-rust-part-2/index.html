The Code

You can find the source code I am building in this series over at github. Here’s the db module :

use sqlx::{migrate, Pool, Postgres, postgres::PgPoolOptions, FromRow};
use serde::{Serialize, Deserialize};
use anyhow::Result;
use crate::config::CONFIG;

#[derive(Debug, FromRow, Clone, Serialize, Deserialize)]
pub struct UrlMap {
    key: String,
    url: String,
}

impl UrlMap {
    pub fn new(key: String, url: String) -> Self {
        Self { key, url }
    }
}

pub struct DB {
    pub pool: Pool<Postgres>,
}

impl DB {
    pub async fn new() -> Result<Self> {
        let pool = PgPoolOptions::new()
            .max_connections(CONFIG.database.max_connections)
            .connect(&CONFIG.database.url)
            .await?;

        let migrator = migrate!();
        migrator
            .run(&pool)
            .await?;

        Ok(Self { pool })
    }
}
use anyhow::Result;
use crate::config::CONFIG;
use crate::db::{DB, UrlMap};

mod config;
mod db;

#[tokio::main]
async fn main() -> Result<()> {
    println!(
        "host: {}, port: {}, database.url: {}",
        CONFIG.host, CONFIG.port, CONFIG.database.url
    );

    let db = DB::new().await.unwrap();
    let res = sqlx::query_as::<_, UrlMap>("select * from url_maps")
        .fetch_all(&db.pool)
        .await?;

    println!("results: {:?}", res);

    Ok(())
}

The Gist

We create a struct DB to hold a reference to a connection pool for Postgres. Within the constructor we utilize sqlx apis to instantiate a connection pool using the configurations from our config model we setup previously in Part I. We also execute our migrations automatically. SQLX ensures only migrations that haven’t been executed previously are executed.

We also define UrlMap , our data model which represents our data for mapping akey to a url within the url_maps table.

We instantiate an instance of DB struct in our main function and use the connection pool reference from that to execute a query to fetch data from the url_maps table and print it out to the console.

This sets up the database for our Url Mapper application, that we’re building as part of this series and we’ll be utilizing this to build our service and a CRUD API in the coming episodes.

Conclusion

​ We learn how to utilize sqlx apis to do the following :

  • Establish connection to the database, setup a connection pool for performance
  • Build a modelUrlMap to represent data

  • Execute query and parse the results as UrlMap type ​
