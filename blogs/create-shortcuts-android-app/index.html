How to create shortcuts in your Android app

Raja Jawahar

Mobile Developer

One of the fun parts of working on a product for more than half a decade is that I’ve always thought of ways to add cool features in the app, like shortcuts and WidgetKit.

This blog will give you a walkthrough of how I have implemented Android shortcuts in our app.

A brief about Android shortcuts

Shortcuts are features (quick services) that help users quickly start some everyday or recommended tasks within the app. Long-pressing on the app’s launcher icon will show the shortcuts defined on the app.

There are three kinds of shortcuts: Static, Dynamic, and Pinned shortcuts.

Static Shortcuts

This type of shortcut is used for common user activities. In the screenshot above, Google has used static shortcuts — ‘New Tab’ and ‘Incognito Tab’. We can use this shortcut type if the app does not have any kind of configuration-based action or authentication needed to launch an action.

This can be implemented by adding a single resource file.

How to create static shortcuts:

  1. Select your resource folder

  2. File -> New ->XML -> Name your file as shortcuts

  3. Define your shortcuts in xml.
  4. In the below example, I am creating a static shortcut for navigating to Create Work Order screen on an Asset Management app.
<shortcuts xmlns:android="http://schemas.android.com/apk/res/android">
  <shortcut
    android:shortcutId="create_workorder"
    android:enabled="true"
    android:icon="@drawable/ic_shortcut_create_wo"
    android:shortcutShortLabel="@string/compose_shortcut_work_order_short_label"
    android:shortcutLongLabel="@string/compose_shortcut_work_order_long_label"
    android:shortcutDisabledMessage="@string/compose_work_order_disabled_message1"
    >
    <intent
      android:action="android.intent.action.VIEW"
      android:targetPackage="com.sedintechnologies.workbook.dev"
      android:targetClass="com.sedintechnologies.workbook.view.activity.CreateWoActivity"
      />
  </shortcut>

</shortcuts>
  1. Include this in your activity tag in AndroidManifest.xml, which has intent filter category as LAUNCHER.
<activity
  android:name=".view.activity.SplashWithLoginActivity"
  android:exported="true"
  android:label="@string/app_name"
  android:theme="@style/AppTheme.SplashScreen">
  <intent-filter>
    <action android:name="android.intent.action.MAIN" />
    <category android:name="android.intent.category.LAUNCHER" />
  </intent-filter>

  <meta-data
    android:name="android.app.shortcuts"
    android:resource="@xml/shortcuts" />
</activity>

Dynamic ShortCuts

Dynamic shortcuts are context-sensitive; we can implement this based on user actions. This can be changed based on the uses of the app.

A good example is calling a specific person in the Contacts app and navigating to a specific location in Google Maps. XML is not necessary for this one.

In the below example, I have added the code snippet of the dynamic shortcut, which I have added for our product app. Our app has a page called Work Order List. Whenever users open the work order, we create a shortcut to open the specific work order, and this will vary based on the customer.

private void setShortCutsForWorkOrder() {
  Intent intent = new Intent(this, WorkOrderDetailActivity.class);
  intent.putExtra("workOrderId", workorder.getWorkOrderId());
  intent.putExtra("isReadOnly", isReadOnly);
  intent.setAction(Intent.ACTION_VIEW);
  ShortcutInfoCompat shortcutInfoCompat = new ShortcutInfoCompat.Builder(this, "open_workorder")
      .setShortLabel("WorkOrder #"+ workorder.getWorkOrderNum())
      .setLongLabel("Open the WorkOrder #"+ workorder.getWorkOrderNum())
      .setIcon(IconCompat.createWithResource(this, R.drawable.ic_shortcut_view_wo))
      .setIntent(intent).build();
  ShortcutManagerCompat.pushDynamicShortcut(this, shortcutInfoCompat);
}

Since we are using the common id as open_workorder in our shortcut builder, when users open the item in the list, the shortcut will automatically be updated.

Pinned Shortcuts

Pinned shortcuts are used for user-driven actions. These shortcuts appear as a different launcher icon on the screen. They require Android 9 (API level 28) and above. There is no limit to the number of pinned shortcuts we can create. For example, a user might want to pin a recent map to the Home screen. This is an advantage because it reduces multiple steps.

This can be done in two ways.

  • By long-pressing on the specific shortcut, we are able to create a pinned shortcut. Pinned shortcut for Android

  • Updating the existing dynamic shortcut with its ShortcutId.

    private void createPinnedShortCut() {
  ShortcutManager shortcutManager = getSystemService(ShortcutManager.class);
  if (shortcutManager.isRequestPinShortcutSupported()) {
    ShortcutInfo pinShortcutInfo =
        new ShortcutInfo.Builder(this, "open_workorder").build();
    Intent pinnedShortcutCallbackIntent =
        shortcutManager.createShortcutResultIntent(pinShortcutInfo);
    PendingIntent successCallback = PendingIntent.getBroadcast(this, /* request code */ 0,
        pinnedShortcutCallbackIntent, /* flags */ 0);
    // If your target version is more than 31+, use PendingIntent.FLAG_IMMUTABLE flag
    //    PendingIntent successCallback = PendingIntent.getBroadcast(this, /* request code */ 0,
    //           pinnedShortcutCallbackIntent, PendingIntent.FLAG_IMMUTABLE);
    shortcutManager.requestPinShortcut(pinShortcutInfo,
        successCallback.getIntentSender());
  }
}

But there are some limitations:

  • The minimum SDK is 25.
  • A maximum four shortcuts are visible.
  • API allows a combination of 15 shortcuts (both Static and Dynamic)
  • Short Description Character Limit: 10.
  • Long Description Character Limit: 25
  • App Shortcuts Design Guidelines will apply.
